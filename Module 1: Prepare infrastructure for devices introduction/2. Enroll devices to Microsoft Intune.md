# Device Enrollment

## Prerequisites (Must-Haves)

These are mandatory for enrollment to succeed:

- **Intune License**: Assigned via Microsoft 365 E3/E5, EMS E3/E5, or standalone Intune.
- **Microsoft Entra ID Tenant**: All users and devices must exist in the tenant.
- **MDM Authority**: Intune must be set as the Mobile Device Management authority.
- **Verified Custom Domain**: Enables branded UPNs (e.g., `user@yourcompany.com`).
- **Supported OS Versions**:
  - iOS/iPadOS 15.0+
  - macOS 13.0+
  - Android 8.0+, Android Enterprise, AOSP
  - Windows 10/11 (Home, S, Pro, Education, Enterprise, IoT)
  - Windows 10/11 Cloud PCs (Windows 365)
  - Windows 10 LTSC 2019/2021
  - Surface Hub
  - Ubuntu Desktop 20.04 / 22.04
- **Network Access**: Devices must reach Intune and Entra endpoints.

## Windows Enrollment
Windows Enrollment is the process of registering and managing Windows devices in Intune so that IT can apply policies, apps, and compliance rules.

### Enrollment Methods

| Enrollment Method | Ownership Type | Best For | Description |
|--------------------|----------------|-----------|--------------|
| **Windows Autopilot** | Corporate | New devices (OOBE setup) | Simplifies deployment by preconfiguring devices from the vendor to automatically enroll into Intune. |
| **Azure AD Join (Cloud-only)** | Corporate | Remote or cloud-first environments | Device joins Azure AD directly and automatically enrolls in Intune. |
| **Hybrid Azure AD Join** | Corporate | On-prem AD + Intune environments | Devices join on-prem AD, then sync to Azure AD and enroll into Intune. |
| **Bulk Enrollment (Provisioning Package)** | Corporate / Shared | Kiosks, labs, or shared devices | Use Windows Configuration Designer to create a provisioning package (*.ppkg*) for offline or mass enrollment. 
| **Co-management (SCCM + Intune)** | Corporate | Transition phase from SCCM to Intune | Devices managed by SCCM can also be connected to Intune for cloud-based management. |
| **Manual Enrollment (Settings app)** | BYOD / Personal | Personal devices or test devices | User goes to *Settings â†’ Accounts â†’ Access work or school â†’ Connect*, and signs in with company credentials. |
| **Windows Autopilot for existing devices** | Corporate | Devices already in use | Used to reset and reconfigure existing devices for new users while keeping enrollment automated. |
